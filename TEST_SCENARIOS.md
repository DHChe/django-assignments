# Test Scenarios

이 문서는 현재 Todo 프로젝트를 사용자 여정과 예외 변수 관점에서 검증하기 위해 정리한 테스트 시나리오다.

## 1. 사용자 여정 기반 시나리오

| 단계 | 시나리오 | 기대 결과 |
| --- | --- | --- |
| 비회원 진입 | 비로그인 사용자가 `/todo/`에 접속한다. | 서비스 소개와 로그인/회원가입 CTA가 보이고, 개인 Todo 데이터는 노출되지 않는다. |
| 회원가입 | 새 사용자가 올바른 정보로 가입한다. | 계정이 생성되고 로그인 화면으로 이동한다. |
| 로그인 | 기존 사용자가 올바른 자격 증명으로 로그인한다. | Todo 목록 화면으로 이동한다. |
| Todo 생성 | 로그인 사용자가 제목, 설명, 시작일, 종료일을 입력해 Todo를 생성한다. | Todo가 로그인 사용자 소유로 저장되고 상세 화면으로 이동한다. |
| 목록 확인 | 로그인 사용자가 목록 화면에 진입한다. | 자신의 Todo만 보여야 하고, 다른 사용자 Todo는 보이면 안 된다. |
| 검색 | 사용자가 제목 또는 설명 키워드로 검색한다. | 해당 키워드와 일치하는 Todo만 보여야 한다. |
| 상세 보기 | 사용자가 자신이 작성한 Todo 상세 화면에 진입한다. | 제목, 설명, 일정, 완료 상태, 수정/삭제 액션이 보여야 한다. |
| 수정 | 사용자가 Todo 내용을 수정하고 완료 상태를 바꾼다. | 수정 내용이 저장되고 상세 화면에 반영된다. |
| 삭제 | 사용자가 Todo를 삭제한다. | 목록 화면으로 이동하고 해당 Todo는 DB에서 제거된다. |

## 2. 변수 상황 기반 시나리오

| 분류 | 시나리오 | 기대 결과 |
| --- | --- | --- |
| 인증 | 비로그인 사용자가 생성/상세/수정/삭제 URL에 직접 접근한다. | 로그인 페이지로 리다이렉트된다. |
| 권한 | 사용자가 다른 사람의 Todo 상세/수정/삭제 URL에 접근한다. | 404가 반환되어 데이터 존재 여부가 노출되지 않는다. |
| 필수값 누락 | Todo 생성 시 필수 입력값을 비워 둔다. | 폼 에러가 노출되고 데이터는 저장되지 않는다. |
| 일정 역전 | 종료일이 시작일보다 빠른 Todo를 생성/수정한다. | 날짜 검증 에러가 노출되고 데이터는 저장되지 않는다. |
| 검색 결과 없음 | 검색어와 일치하는 Todo가 없다. | 빈 상태 UI가 보이고 서버 에러가 발생하지 않는다. |
| 페이지네이션 | 10개를 초과하는 Todo가 존재할 때 2페이지로 이동한다. | 2페이지 데이터만 보여야 하고 총 페이지 수가 올바르게 계산된다. |
| 삭제 요청 방식 | 삭제 URL을 GET으로 호출한다. | 405 응답이 반환되고 데이터는 삭제되지 않는다. |
| 로그인 실패 | 비밀번호가 틀린 상태로 로그인한다. | 에러가 노출되고 세션은 생성되지 않는다. |
| 회원가입 실패 | 비밀번호 확인이 일치하지 않는다. | 계정이 생성되지 않고 회원가입 화면에 머문다. |

## 3. 자동화 테스트 반영 범위

- `users/tests.py`: 회원가입/로그인 정상 흐름과 실패 흐름 검증
- `todo/tests.py`: 목록, 검색, 페이지네이션, 상세, 생성, 수정, 삭제, 인증, 권한, 날짜 검증 검증
